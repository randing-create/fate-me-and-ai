# 项目交接说明 · 给 Cowork 的第一条上下文

这是一份让新会话（Cowork）快速接手本项目的说明。请先完整读一遍本文件，再读 `bible.md`（视觉圣经）和 `pilot-notes.md`（音乐锚点分析），之后就具备了和之前 Chat 会话同等的项目理解。

## 这是什么项目

把贝多芬第五交响曲《命运》重新讲成一个宇宙寓言，用纯代码生成的动态视觉（SVG + HTML + 内嵌音频）逐秒对齐音乐结构。创作者：Ran D。这不是"给音乐配画面"，是把 Ran D 听到的一条内在叙事——受困→挣扎→扎根→回望→点亮别人——用色彩代数和镜头语言精确编排出来。

核心 logline：**她害怕的那股力量，有一滴落进了她心里。她把它养大了。**

## 故事骨架（已锁定，勿更改核心设定）

- **第一乐章**（zoom out）：暗黑宇宙中，暖光星球独立于黑洞与乱流之间。发展部埋种子：一滴深紫暗流脱离主流坠入星球，无人察觉。尾声：光收拢向内，扎根找到中心，乱流从对抗变为环绕的漩涡，达到平衡。
- **第二乐章**（zoom in，flashback，尚未制作）：壳内的原初世界——河流、水草、鱼群、大象一家、鸟群，最后是年轻的师父（女性），奔跑于山野，登高回望，大地充满爱与希望。
- **第三乐章**（尚未制作）：种子发芽。小精灵（暗物质所生）滑稽登场，与女孩互为师徒，学百家技能，被爱浇灌染上橙粉色。捉迷藏走丢，回到坠落之地——橙粉黯下去，深紫透出来，远方出身的声音在召唤。
- **第四乐章**（尚未制作）：三→四乐章不间断连奏(attacca)。打开的瞬间紫与粉同时绽放——不是驯化，是整合。回到宇宙，点点星光引发远方星球共振连桥；遇见深紫色的同类，从试探到华尔兹，粉紫烟花照亮夜空。

## 已完成：第一乐章完整版

- 文件：`mvt1-full.html`（约8MB，音频以 base64 内嵌，打开即可全屏播放，7分20秒）
- 制作方法：用 librosa 对音频做 RMS 能量包络分析 + 起音检测，取得精确到 ±0.2 秒的音乐锚点（命运动机、强奏、静默豁口、圆号召唤等），再用一个纯 JS 引擎按时间轴驱动 SVG 场景（星球位置、光的强弱、乱流速度、镜头推拉全部是时间的函数，部分直接读取音频能量值驱动，比如尾声根须的生长速度对应锤击的力度）。
- 音乐结构对位：呈示部反复（两遍，第二遍更暗，象征"失去的土地没有回来"）→ 发展部（风暴、暗滴坠入种子）→ 再现部（第三次叩门唤醒壳内的紫斑，双簧管独白）→ 尾声（扎根、点亮中心、乱流化为有序漩涡）。
- 音乐授权：Fulda Symphonic Orchestra / Simon Schindler 演奏，来源 classicals.de，CC BY-SA 4.0，需保留 credit（已写入片头/片尾）。

## 制作方法论（后续乐章复用这套流程）

1. **锁定该乐章的音源片段**（同一版录音）
2. **用 Python + librosa 做能量包络分析**：提取 RMS dB 曲线、onset 检测，找出该乐章的关键锚点（强弱变化、主题进入点、静默/延长记号、速度变化）
3. **写分镜表**：锚点 ↔ 画面事件 ↔ 情绪，参考 `pilot-notes.md` 的格式
4. **写 HTML/SVG/JS 引擎**：一个 `update(t, dt)` 函数按当前播放时间驱动所有视觉元素，音频用 `<audio>` 标签的 `currentTime` 作为唯一时间源，画面绝不能自己计时（否则会和音乐失步）
5. **本地全时间线模拟测试**（用 Node.js 跑一个 DOM stub，遍历整个时长调用 `update()`，确保没有运行时报错）再交付
6. **每次修改优先问 Ran D 两个问题**：哪一秒起鸡皮疙瘩，哪一秒出戏——这是校准视觉语言最有效的反馈方式
